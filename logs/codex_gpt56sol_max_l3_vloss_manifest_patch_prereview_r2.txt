## Findings

1. **P1 — The new 500-step pilot cannot produce evidence accepted by its verifier.** The launcher invokes training directly without creating `logs/hy273_l3_vloss_jit_bound_pilot500_v2.log` and omits `--log_every 1` ([pilot launcher](/mnt/afs/mogeflow-control/scripts/launch/train_hy273_raw_flow_l3_vloss_pilot_ddp4.sh:104)). Training therefore uses the default `log_every=20` ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:117)) and emits only step 1 plus multiples of 20—26 records total ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1604)). The verifier requires exactly steps 1–500 ([verify_hy273_l3_vloss_preflight.py](/mnt/afs/mogeflow-control/tools/verify_hy273_l3_vloss_preflight.py:198)) at a fixed log path ([verify_hy273_l3_vloss_preflight.py](/mnt/afs/mogeflow-control/tools/verify_hy273_l3_vloss_preflight.py:31)). Additionally, `RUN_NAME` remains environment-overridable while the verifier’s run directory is fixed ([pilot launcher](/mnt/afs/mogeflow-control/scripts/launch/train_hy273_raw_flow_l3_vloss_pilot_ddp4.sh:93)). As shipped, a direct pilot launch consumes 500 GPU steps but cannot pass the mandatory gate.

2. **P2 — The legacy `loss_fork` exception is not conditional on the checkpoint actually being legacy.** Any `loss_fork` request receives the missing-field allowlist ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1391), [train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1405)); validation does not inspect the saved contract version before applying it ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:730)). A direct probe confirmed that a checkpoint claiming `resume_contract_version=2` but missing all three v-loss/source fields is accepted in `loss_fork`. This does not affect the scratch pilot, but weakens the stated v2/legacy distinction.

3. **P2 — Resume SHA validation has a hash/load TOCTOU window.** Rank 0 hashes the path at startup ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1301)), but the checkpoint is reopened only after dataset and model construction ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1382)). A concurrently replaced checkpoint could therefore differ from the verified bytes. Stable-file validation works correctly; a deliberately wrong SHA was rejected with the expected and actual digests.

4. **P2 — The pilot/production environment is not attested.** The source list covers code, calibration, metadata, and payload manifests but no interpreter/package/CUDA fingerprint ([manifest builder](/mnt/afs/mogeflow-control/tools/build_hy273_l3_vloss_source_manifest.py:12)). The trace contract records device IDs and port, not Python/PyTorch/CUDA/cuDNN versions ([train_hy273_raw_flow.py](/mnt/afs/mogeflow-control/train_hy273_raw_flow.py:1465)). Updating the hard-coded `mogo` environment between pilot and production would not invalidate the gate.

**Explicit verdict: no P0 remains, but one P1 remains. The 500-step pilot is NO-GO until Finding 1 is fixed.**
